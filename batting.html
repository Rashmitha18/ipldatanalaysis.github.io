Prediction of Batsman Scores


Here according to the hackathon we have data from 2008 to 2017 and 2018 data. In 2018 it has only schedule of the matches like venue id, teams and players. Things I did is for predicting the 2018 players score for each match I filtered only the players who are playing 2018 from data set 2008 to 2017. In data sets we have ball_by_ball data which has each and every ball bowled since 2008 to 2017. I grouped it to get the players total score in each match in every season. Next calculated cumulative sum of runs and matches played so that I can calculate the overall strike rate and average of players. Then I thought that very first match of a player in IPL is not required since he don’t have any records before so that I shifted the strike rate, average and total matches played to the next match he played. Thinking that any player before playing any match we have the predictor variables like his Venue, Opponent team, Number of matches he played, overall strike rate, average for all his previous matches in Ipl. Then shifted the strike rate, average, matches played of player for very last match he played to 2018 data as a predictor. Used 2017 matches as the validation data for model. Used 2018 matches for test set. I used to different models like Lasso which is a regularized linear predictor and Gradient Boosting Regressor both got approximately equal Error rate. This is completely brief explanation of what we did for modelling. We used python and Jupyter notebook for the complete analysis. Data sets and the notebooks are placed in Github.
